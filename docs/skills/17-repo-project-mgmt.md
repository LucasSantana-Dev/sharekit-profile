# Repository & Project Management Skills

Use active skills for repo intake, backlog shaping, issue creation, release readiness, and health checks. Archived wrappers such as `onboard-new-repo` and `audit-deep` are represented by active skill sequences.

---

## First-touch repo pattern

1. `/session-bootstrap` to rehydrate context and identify blockers.
2. `/context-pack` for project standards and prior decisions.
3. `/quality-assurance` to choose health checks.
4. `/quality-gates` for repository-native verification.
5. `/knowledge-loop` to capture durable repo-specific decisions.

---

## /next-priority

Decide the highest-value safe thing to do now based on blockers, PRs, issues, risk, and effort.

---

## /pr-merge-readiness

Aggregate merge signals for an open PR: required checks, reviews, branch protection, conflicts, and release risk.

---

## /ci-watch

Monitor CI, identify failing required checks, and summarize blockers.

---

## /changelog-update

Promote unreleased changes into a changelog entry before release.

---

## /version-bump

Bump project version consistently with release policy.

---

## /ship

Release-ready shipping workflow once required gates pass.

---

## /dep-sweep

Dependency maintenance and risk review for updates.

---

## Backlog pattern

Use `/quality-assurance` for audit scope, `/next-priority` for ranking, `/plan` for larger work, and GitHub/project tooling for issue creation.
