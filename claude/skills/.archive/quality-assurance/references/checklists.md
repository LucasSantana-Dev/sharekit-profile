# QA Checklists

## Pre-merge

- code quality and build confidence,
- tests for the changed behavior,
- security checks if the surface changed,
- docs or migration notes when the interface changed.

## Release

- full project verification,
- security posture review,
- targeted regression or smoke plan,
- release-specific evidence package.

## Maintenance sweep

- dependency and security status,
- recurring validation gates,
- unresolved risk inventory.
