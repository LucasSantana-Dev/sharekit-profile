Core concepts

Eight ideas that explain everything

Skill
A reusable, well-scoped workflow invoked with /name. Runs in your current context; most are read-only and hand you findings to apply.
e.g. /code-review reads your diff → returns severity-rated findings.
Composite
A skill that auto-chains several sub-skills with a gate between phases. The router fires 🎯 Composite match when your intent matches one. Always prefer it — running sub-skills by hand skips critical phases.
e.g. /refactor-pipeline → plan → 3-agent team → test-cleanup → ADR → docs-sync.
Sub-agent
An autonomous Claude in its own context with a scoped toolset. Analysis agents (Explore, critic) are read-only by agentType — they structurally can't write. Dispatch many in parallel.
e.g. /phase-runner spawns one agent per task, one wave at a time.
Hook
A shell script wired to a lifecycle event (session start/end, every prompt, every tool call). Fires automatically — exit 0 = silent, no overhead. You never invoke them.
e.g. on SessionEnd → mirror ~/.claude to cloud backup.
Standard
A policy doc the operator loads on demand — naming, testing, security, decision discipline. Standards are the "house rules" skills and agents follow.
e.g. code-standards.md, security.md, red-flags.md.
Model tiering
Match model strength to task. Haiku for mechanical work, Sonnet for implementation, Opus for orchestration & architecture. A hook hints the tier each turn.
e.g. /smart-model-select before any multi-agent job.
Autonomy
Default is proceed and report, not pause and ask. Sub-decisions inside delegated scope are the operator's to make. It only stops for destructive, production, or out-of-scope actions.
e.g. "improve the tests" → it picks which tests & targets, then proceeds.
Caveman mode
Default-on terse style: drops articles & filler, keeps every technical term, command, and quoted error verbatim. Faster to read, cheaper in tokens.
e.g. "PreToolUse hook blocks rm -rf → surfaced, not run."

Under the hood

What happens when you hit enter

Every prompt runs through the hook pipeline before the operator even starts thinking — 9 lifecycle events, 36 scripts, all silent on success.

SessionStartOnce. Pull memories + ADRs from the env repo, reindex drifted RAG chunks, warn on branch drift or oversized memory.
UserPromptSubmitEvery prompt. Auto-recall injects a # Knowledge graph context block · classify complexity → hint model tier · emit 🎯 Composite match if intent fits · warn if context > 85%.
PreToolUseSafety gate. Block dangerous bash (rm -rf, sudo rm), writes to protected paths, and redundant re-reads.
PostToolUseObserve. Reindex changed files on Write/Edit · warn on large reads or > 3 edits/turn.
SessionEndCleanup. Log token spend, then mirror memory + RAG to persistent cloud storage.
Guardrails

Hard rules the operator won't break

Objective, non-negotiable. They override convenience and even composite behavior.

01Never automate a PR with a human's comments — or any open PR authored by another person. Bots (CodeRabbit, dependabot) don't count.
02Parallel for ≥2 independent tasks — one agent each, own worktree per same-repo agent. Sequential is a contract violation.
03Analysis agents read-only by agentType — structurally write-incapable, not just told "don't write".
04State-check before mutation — query current state first; skip if already satisfied. Idempotent by default.
05No big-bang rewrites without a demand gate — measure usage first, prototype one unit, then commit.
06Stuck protocol — > 2 attempts without progress → surface it, switch approach, then escalate. Never silently loop.
07Pattern Discovery Protocol (mandatory before implementation) — "Search First, Reuse Always, Create Only When Necessary." Before any non-trivial work: search specs (recall/grep docs/specs/) → search codebase (grep/glob) → search session history (recall) → consult docs → architectural validation (route to @oracle for non-obvious cases). Chain context-pack before plan/scope-and-execute/refactor. Trivial single-file edits (<20 lines, known path) are exempt.
08Independence Gate (review/security/critic agents) — Review, security, and critic agents MUST be independent subagents — never collapsed into the implementer lane. Roles that MUST be independent: code-reviewer, security-reviewer, critic. Read-only-by-construction is about tool permissions (no write/edit). Independence is about lane separation — even with no write tools, a reviewer running in the same context as the implementer has compromised objectivity. Spawn an independent review subagent: after @fixer completes → @code-reviewer or @oracle; before merge → @code-reviewer; security-sensitive → @security-reviewer; high-stakes → critic.
